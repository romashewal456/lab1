Лабораторна робота №1: Рефакторинг та ООП-проєктування гри "Хрестики-Нулики"

📝 Опис проєкту
Цей проєкт є випускною роботою в межах Лабораторної роботи №1, де було виконано перехід від базового процедурного коду гри "Хрестики-Нулики" (Tic-Tac-Toe) до чистої, модульної об'єктно-орієнтованої архітектури (ООП).

 📁 Структура проєкту
Проєкт структуровано відповідно до найкращих практик розробки програмного забезпечення:
* `src/` — вихідний код нової ООП-версії гри.
  * `Board.hpp` — клас ігрового поля (збереження стану сітки, валідація ходів).
  * `GameEngine.hpp` — клас логіки гри (контроль черги, перевірка умов перемоги/нічиєї).
  * `ConsoleUI.hpp` — клас текстового інтерфейсу (виведення стану на екран, зчитування введення).
  * `new_main.cpp` — головна точка входу (оркестратор ігрового циклу).
* `tests/` — автоматизовані Unit-тести для перевірки цілісності логіки `GameEngine`.
* `docs/` — UML-діаграма класів (проєктний файл `.drawio` та експортована картинка `.png`).
* `procedural_version/` — початкова процедурна версія коду для збереження історії рефакторингу.

🛠️ Інструкція зі збирання та запуску

Для компіляції проєкту необхідно мати встановлений компілятор `g++`.

 Запуск гри (ООП версія):
```bash
g++ src/new_main.cpp -Isrc -o oop_game
./oop_game
